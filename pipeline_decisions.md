# Pipeline decisions summary

| # | Decision | Choice | Rejected alternatives | Justification |
|---|----------|--------|-----------------------|---------------|
| 1 | Overall pipeline strategy | Alignment-based (Pipeline 1): STAR → featureCounts → DESeq2 | Pseudoalignment (Pipeline 2): kallisto → sleuth | Biological question is gene-level (KRAS/MAPK reactivation, *EGFR* downregulation, *SLC16A1* upregulation), not isoform-level. DESeq2's GLM framework composes cleanly with the 2 × 3 factorial design. No indication of unusually high quantification uncertainty — paired-end Illumina reads, established human cell lines, well-annotated reference. Tradeoff accepted: STAR alignment costs ~15–20 min/sample vs ~1 min/sample for kallisto, but the BAM output is retained for downstream inspection (e.g., confirming the *KRAS* activating mutation in LIM1215-R). |
| 2 | QC tool | FastQC (v0.12.1) | — | Course-standard QC tool; produces the per-module diagnostics (per-base quality, adapter content, overrepresented sequences, GC content) needed to drive trimming decisions. |
| 3 | QC sample selection (preliminary) | SRR28499039 (LIM1215-S, rep 1) and SRR28499036 (LIM1215-R1, rep 1) | Random sample pair; one sample per cell line | Mirrors the core biological contrast (sensitive parent vs first independent resistant derivation) within one cell line and one sequencing batch, so cross-condition differences reflect biology rather than batch. Full analysis would run FastQC on all 18 samples and aggregate with MultiQC. |
| 4 | Adapter / poly-G / quality trimming tool | fastp | Trimmomatic; cutadapt (+ separate poly-G pass) | Trimmomatic has no native poly-G trimming, which is disqualifying given the poly-G tails observed in SRR28499036 reverse reads (diagnostic of Illumina two-colour chemistry). fastp handles adapter auto-detection and poly-G removal in a single invocation; cutadapt would require a second tool for poly-G. |
| 5 | Quality trimming aggressiveness | fastp defaults only (no aggressive sliding-window trimming) | Sliding-window quality trimming (e.g., Phred 20) | All four QC files showed mean Phred > 30 at every position with no 3′ decay. Aggressive trimming would discard usable signal without improving downstream results. |
| 6 | Minimum post-trimming read length | 36 bp (`--length_required 36`) | No length filter; 50 bp | 36 bp is the commonly cited floor for unique mapping to the human genome. Given 76 bp raw reads, this tolerates modest adapter/poly-G removal while preserving mappability. |
| 7 | Splice-aware aligner | STAR (two-pass mode) | HISAT2; Bowtie2 (ungapped) | Bowtie2 cannot place reads spanning exon–exon junctions (~15–30% of reads at 100 bp). Between the two splice-aware aligners taught in the course, STAR discovers novel junctions *de novo* via maximal mappable prefix seeds, whereas HISAT2 is annotation-guided — relevant for cancer cell lines where splicing may diverge from reference. STAR is also faster at alignment time. Cost: ~30 GB RAM vs HISAT2's ~8 GB, acceptable on a compute cluster. |
| 8 | Reference genome + annotation | GRCh38 primary assembly + GENCODE v45 GTF | UCSC hg38 + RefSeq; earlier GENCODE versions | GENCODE is the standard annotation paired with STAR/featureCounts workflows and is more comprehensive than RefSeq for transcript diversity. Using the same GTF for alignment (`--sjdbGTFfile`) and counting ensures consistent gene/exon definitions. |
| 9 | STAR junction discovery mode | Two-pass (`--twopassMode Basic`) | Single-pass | First pass collects candidate novel junctions across all samples; second pass realigns every read against an augmented index, so reads spanning novel junctions are placed consistently across the cohort. |
| 10 | STAR splice-junction overhang | `--sjdbOverhang 75` | Default (100) | Read length is 76 bp (from FastQC); convention is `read_length − 1`. |
| 11 | Read-counting tool | featureCounts (Subread) | htseq-count; Salmon + tximport (transcript-level) | featureCounts is fast, well documented, and produces an integer gene × sample matrix matching DESeq2's input assumptions. htseq-count is slower with no accuracy advantage at the gene level. Transcript-level quantification was rejected for the same reason as Pipeline 2 — the hypothesis is gene-level. |
| 12 | Counting unit | Paired-end fragments (`-p --countReadPairs`) | Reads counted individually | Data is paired-end; counting each pair once (not twice) is required for correct library size estimation. |
| 13 | Feature aggregation level | Gene-level (`-t exon -g gene_id`) | Transcript-level; exon-level | Biological question is at gene level. Acknowledged limitation: gene-level counts are confounded if isoform usage changes between conditions (Lecture 8), flagged in *Limitations*. |
| 14 | Strandedness | Reverse-stranded (`-s 2`), pending empirical confirmation | Unstranded (`-s 0`); forward-stranded (`-s 1`) | Modern Illumina RNA-seq library preparations (e.g., TruSeq Stranded) are typically reverse-stranded. Value will be confirmed with RSeQC `infer_experiment.py` on a subset of BAMs before locking in, because a wrong strandedness setting drops ~50% of true counts. |
| 15 | Differential expression tool | DESeq2 | edgeR; limma-voom; sleuth | DESeq2 is the course's primary teaching vehicle for the Three Challenges and pairs naturally with the alignment-based pipeline. edgeR would be defensible (similar NB + empirical Bayes framework) but is not the pipeline developed in Lecture 9. |
| 16 | Design formula — shared signature | `~ cell_line + condition` | `~ condition` (pooling cell lines) | Additive model allows the cell-line term to absorb baseline DiFi vs LIM1215 differences so that the condition term isolates the resistance effect common to both contexts. |
| 17 | Design formula — cell-line-specific signature | `~ cell_line * condition` (interaction tested) | Two separate `~ condition` fits, then intersect/difference | Interaction term directly tests whether the resistance response differs by cell line, which is the correct framing for identifying DiFi-R-specific (*EGFR* loss) and LIM1215-R-specific (*SLC16A1*) effects. |
| 18 | R1 vs R2 comparison | Within-cell-line `~ condition` fit, contrast R1 vs R2 | Collapsing R1 and R2 into single "resistant" factor | R1 and R2 are independent resistance derivations; explicit comparison tests whether they converge on the same transcriptional state. Expected: far fewer DE genes than R vs S. |
| 19 | Statistical test | Wald for single-coefficient contrasts; LRT for interaction and multi-level comparisons | Wald for everything | Lecture 9 slide 18 explicitly recommends LRT for complex experimental designs; Wald is appropriate for two-group contrasts. |
| 20 | Multiple-testing correction | Benjamini–Hochberg FDR at 5% | Bonferroni FWER; uncorrected p | ~20,000 genes tested simultaneously. BH controls FDR (expected proportion of false positives among discoveries), the appropriate error rate for gene-expression screens. Bonferroni (FWER) is overly conservative for this many tests. Applied by DESeq2 by default. |
| 21 | Effect-size filter | `|log₂ FC| ≥ 1` on the FDR-significant set (for downstream interpretation only) | FDR alone; more stringent FC cutoff | Ensures reported "hits" have both statistical support and biological magnitude. Applied *after* FDR control, not as a pre-test filter, so it does not affect the p-value distribution. |
| 22 | Log fold change shrinkage | `apeglm` estimator | Unshrunken MLE; `normal` shrinkage | Reduces inflated fold-change estimates from low-count genes without penalising well-supported large effects (Lecture 9, slides 33–34). `apeglm` is the current DESeq2 recommendation, outperforming the older `normal` prior. |
| 23 | Normalisation (Challenge 1) | DESeq2 median-of-ratios size factors | TPM/FPKM; total-count scaling | Median-of-ratios is robust to compositional bias — a single highly expressed gene changing cannot distort size factor estimates because the median is taken across all genes. TPM/FPKM are within-sample units and fail for between-sample comparison. |
| 24 | Variance estimation (Challenge 2) | Negative binomial + empirical Bayes dispersion shrinkage | Poisson (assumes mean = variance); per-gene MLE dispersion | RNA-seq counts are overdispersed, so Poisson underestimates variance. With only 3 replicates per group, per-gene dispersion MLEs are unreliable; shrinkage toward a mean–dispersion trend stabilises estimates (Lecture 9, slides 28–32). |
| 25 | Functional interpretation | GSEA via `clusterProfiler` / `fgsea` against Hallmark, KEGG, Reactome | Over-representation analysis (ORA) on DE gene list alone | GSEA uses the full ranked gene list and detects coordinated small changes that ORA misses after thresholding. ORA results are included as a secondary check. |
| 26 | Known-gene targeted inspection | Direct lookup of *EGFR*, *KRAS*, *BRAF*, *MET*, *ERBB2*, *SLC16A1*, *LDHA*, *LDHB* in DE table | Rely on GSEA enrichment alone | Specific prior hypotheses from the partner's biological-context section deserve direct evaluation independent of whether they reach genome-wide significance. |
